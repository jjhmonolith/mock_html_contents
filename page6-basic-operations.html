기본 연산: 읽기·추가·삭제

연산이 순서에 미치는 영향을 예측해보세요! ⚙️

🔧 연산의 3가지 기본 동작

읽기=확인, 추가=붙이기/끼우기, 삭제=빼기. 각 연산이 리스트 순서에 어떤 영향을 주는지 예측해보세요!

🚌 현재 버스 리스트

1
🚌
버스A
2
🚌
버스B
3
🚌
버스C
현재 3개의 버스가 있습니다. 연산을 선택해보세요!
읽기

👀 2번째 버스 확인하기

리스트의 특정 위치에 있는 버스를 확인합니다. 순서는 변하지 않아요!

예상 결과: 버스B를 확인, 리스트는 [A, B, C] 그대로
추가

➕ 끝에 버스D 추가하기

리스트 맨 끝에 새로운 버스를 추가합니다. 기존 순서는 유지돼요!

예상 결과: [A, B, C, D] - 끝에 D가 추가됨
삭제

🗑️ 2번째 버스 삭제하기

리스트에서 특정 위치의 버스를 제거합니다. 뒤의 요소들이 앞으로 당겨져요!

예상 결과: [A, C] - B가 삭제되고 C가 2번째로 이동
삽입

↩️ 1번째 앞에 버스E 끼우기

리스트 중간에 새로운 버스를 삽입합니다. 기존 요소들이 뒤로 밀려나요!

예상 결과: [E, A, B, C] - E가 첫 번째, 나머지는 한 칸씩 뒤로

🎮 연산 실행

실행 전 예측부터! 각 연산을 클릭해서 예상 결과를 확인하고, 바뀐 순서를 말로 설명해보세요. 삭제할 때 빈칸이 남지 않고 요소가 앞당겨지는 것을 주목하세요! 📝
